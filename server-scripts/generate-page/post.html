About

A self-authenticating domain name is one that incorporates into itself the information needed to authenticate a connection to the domain.

In practice, the self-authenticating address is (primarily) a cryptographic public key that verifies a cryptographic signature over information describing connection information.

A self-authenticating traditional (SAT) domain name, therefore, begins as a traditional domain (such as example.com) and somehow binds to itself everything needed for self-authentication. For example, prepending a self-authenticating domain as a subdomain of a traditional domain.

If http://{onion}.onion is the address of a Tor onion service, then {onion}.onion is a self-authenticating domain.

As such, {onion}onion.{origin} is a self-authenticating traditional domain because a traditional domain is bound to a self-authenticating domain (binding {origin} with {onion}).

A self-authenticating address (SATA) is a URL joining a traditional address (http://example.com/foo/bar) and a self-authenticating domain. They are joined either:

  • as a SAT domain (as described above)
  • by appending the self-authenticating address in a query string

Extending the previous example, a SATA of https://{origin} is https://{origin}?onion={onion}.

Regardless of which SATA format is used, the SAT domain format is considered canonical form.
